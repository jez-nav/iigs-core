# Apple IIGS Emulator Specification: CPU 65C816

## Feature: CPU programmer-visible state

**Expected behavior:** Implement a WDC 65C816-compatible CPU with 24-bit program addressing, 16-bit accumulator and index registers with 8-bit submodes, emulation/native mode, separate program bank and data bank registers, a 16-bit direct-page register, and a 16-bit stack pointer. In emulation mode, `M` and `X` are forced to 1, the stack is constrained to page `$01xx`, and interrupt stack frames use 6502-compatible width.

**Registers affected:** `A`, `B` accumulator high byte, `X`, `Y`, `S`, `D`, `DBR`, `PBR`, `PC`, and `P` bits `E`, `N`, `V`, `M`, `X`, `D`, `I`, `Z`, `C`.

**Memory addresses involved:** Stack page `$00/0100`..`$00/01FF` in emulation mode; native stack at `$00/0000`..`$00/FFFF`; reset vector `$00/FFFC`; native vectors in bank `$00/FFE4`..`$00/FFFF`; emulation vectors in bank `$00/FFF0`..`$00/FFFF`.

**Edge cases:** `XCE` exchanges carry with emulation mode and immediately forces emulation invariants when entering emulation. `REP`/`SEP` changing `X` from 0 to 1 truncates `X` and `Y` to 8 bits. Direct-page addressing takes an extra cycle when `D` low byte is nonzero. Direct indexed wrap behavior differs in emulation mode when `D` low byte is zero.

**Known compatibility notes:** Some Apple II software relies on the NMOS 6502 indirect JMP page-wrap bug. A IIgs compatibility mode should optionally reproduce the bug for emulation-mode `JMP ($xxFF)` if selected by the target environment. Native 65C816 code expects correct bank wrapping of `PC` within `PBR`, while absolute addressing uses `DBR`.

**Test cases:**

- Reset. Expected: `E=1`, `M=1`, `X=1`, `I=1`, `D=0`, `DBR=0`, `S=$01FF`, `PC=[$FFFC]`.
- Execute `CLC; XCE`. Expected: native mode, `E=0`, previous `E` becomes carry.
- Execute `REP #$30` then load 16-bit `A`, `X`, `Y`; execute `SEP #$10`. Expected: `X` and `Y` high bytes are discarded, `A` remains width selected by `M`.
- Direct-page read with `D=$1201`. Expected: address is `$1201 + operand`, low-16 wrapping, plus one direct-page alignment cycle.

## Feature: Instruction execution and exceptions

**Expected behavior:** Implement all documented 65C816 opcodes, including block moves `MVN` and `MVP`, stack-relative modes, long addressing, `BRL`, `PER`, `PEA`, `PEI`, `WAI`, `STP`, `COP`, `BRK`, `RTL`, `RTI`, and decimal `ADC`/`SBC`. The CPU must charge memory cycles for operand fetches, data reads, data writes, page-cross dummy reads, direct-page alignment penalties, and read-modify-write bus activity.

**Registers affected:** All CPU registers and status flags; `DBR` changes during block moves to the destination bank operand; `PBR` changes on long jumps/calls/returns and interrupts.

**Memory addresses involved:** Opcode fetch from `PBR:PC`; data accesses through direct page, stack page, `DBR`, absolute long, and block-move source/destination banks; vector table in bank `$00`.

**Edge cases:** `BRK` and `COP` increment `PC` past the signature byte before stacking. In native mode interrupts stack `PBR`, `PC`, and `P`; in emulation mode they stack only 16-bit `PC` and `P`. `RTI` restores `P` and the correct-width `PC` frame according to the mode in the restored `P`. `WAI` must resume when a maskable interrupt becomes pending, even if `I` prevents taking it immediately depending on CPU semantics used.

**Known compatibility notes:** Decimal-mode `ADC`/`SBC` must set `N`, `V`, `Z`, and `C` per 65C816 behavior, not simplistic BCD arithmetic. Read-modify-write instructions perform real bus reads/writes that can trigger soft-switch side effects.

**Test cases:**

- `BRK #$12` in emulation mode. Expected: stacked return points after signature; pushed `P` has break bit set for BRK; vector fetched from emulation BRK vector.
- IRQ in native mode. Expected: `PBR`, `PC`, and `P` stacked, `D` flag cleared, `I` set, `PBR=0`.
- `MVN src,dst` with `A=0`. Expected: one byte copied, `A=$FFFF`, `X/Y` incremented, `DBR=dst`, instruction completes.
- `INC $C050`. Expected: I/O read/write sequence is visible, including soft-switch effects.
